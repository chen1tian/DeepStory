## 当前游戏状态
{previous_state}

## 新的聊天记录
{new_messages}

重要规则：
- 只返回 JSON，不要有其他文字、解释或 markdown。
- character_updates 只列出状态有变化的角色；新出现的角色也要列出。
- 主角必须标记 "is_protagonist": true，且主角通常跟随玩家当前位置，presence 为 "present"。
- presence 只能是 "present"、"away" 或 "unknown"：
  - present：角色当前就在 scene_changes.location / 当前场景。
  - away：角色仍存在于故事中，但不在当前场景，留在其他地点或已离开。
  - unknown：剧情没有说明角色在哪里。
- 当玩家切换地点、进入新房间、离开某处、传送、上车/下车、进入/离开建筑时，必须填写 scene_changes.location，并重新给出当前场景真实在场的 scene_changes.npcs。
- scene_changes.npcs 是“当前地点在场 NPC”的完整列表，不要把留在旧地点、没有跟随玩家、只是曾经出现过的人放进去。
- 如果某角色明确留在旧地点或离开当前场景，请在 character_updates 中写出该角色的 location 和 presence: "away"。
- 如果某角色跟随玩家来到新地点，请同时出现在 scene_changes.npcs 中，并可在 character_updates 中标记 location 和 presence: "present"。
- location 表示主要地点，sub_location 表示房间、街角、楼层、洞穴深处等更细的位置。
- relationship_metrics 只更新“当前游戏状态 relationship_metric_configs 中已经为该角色配置过的字段”，例如好感度、信任度、恐惧值。
- 关系数值变化必须来自新聊天记录中的明确互动、态度变化、帮助、伤害、承诺、背叛、亲密行为、冲突或重要共同经历；不要因为普通同场或寒暄自动大幅变化。
- 优先使用 value_delta 表示本轮增减，轻微变化通常为 -3 到 +3，明显帮助/冒犯为 -10 到 +10，重大救命/背叛才使用更大幅度。
- 如果剧情明确给出某关系字段的最终数值，可以使用 value；否则不要猜最终值。
- inventory_changes.action 只能是 add/remove/use/update；category 只能是 equipment/consumable/key_item/material。
- scene_changes.objects 中的 interactable 物品，尽量补充 3-5 个 actions 供前端直接展示与点击，不要所有动作都停留在“看一眼”。
- actions 必须根据“物品本身 + 当前周围环境”联合生成，环境包括：scene_changes.location / sub_location、同场景其他 objects、exits、npcs、以及聊天记录中提到的附近结构、遮挡物、容器、机关、光源、痕迹、声响等。
- actions 必须是“玩家现在就能执行的完整短句”，直接包含目标物，不要只写空泛动词。例如写“查看古石碑背面的字迹”，不要只写“查看”或“调查”。
- 为同一物品设计 actions 时，优先混合这几类动作：1) 观察细节；2) 触碰/试探；3) 改变物品状态或位置；4) 借助周围环境互动。至少给出 1 个不只是观察的动作。
- 如果周围环境提供了明确互动支点，至少给出 1 个“和环境有关”的动作。例如：
  - 物品贴在石碑上，可以写“揭下兔毛贴纸的一角看看下面压着什么”。
  - 物品卡在门缝里，可以写“把卡在门缝里的木片抽出来试试门能不能推开”。
  - 物品放在桌上且附近有烛台，可以写“把信纸凑近烛台看看有没有隐字”。
  - 物品掉在水边，可以写“拿树枝拨近水边的铜钥匙”。
- 动作要符合当前场景逻辑，不能生成玩家此刻做不到的动作。不要凭空引入背包里没有的工具、场景里不存在的机关、或需要远处资源才能完成的行为。
- 动作要尽量具体到目标和意图，而不是泛泛而谈。优先写“揭下兔毛贴纸的一角看看下面压着什么”，不要写“处理兔毛贴纸”。
- 如果聊天内容或当前状态已经暗示某个物品与附近环境存在关系，actions 必须体现这种关系，而不是只重复独立观察动作。
- 不要机械照搬示例；每个物品的 actions 都必须根据该物品当前描述和周围环境重新推断。
- 动作要结合物品当前状态、材质、位置和附着关系来写：
  - 贴着、绑着、卡着、盖着别的东西的物品，应优先考虑“揭下 / 解开 / 抽出 / 移开 / 掀开”之类动作。
  - 容器、门、抽屉、包裹、盒子，应优先考虑“打开 / 拉开 / 拧开 / 翻找”之类动作。
  - 可拿取的小物件，应优先考虑“拿起 / 捡起 / 收入口袋 / 塞进包里”之类动作。
  - 固定在场景中的机关或摆件，应优先考虑“按下 / 推动 / 旋转 / 敲击 / 对准 / 挪开”之类动作。
- 如果描述里出现明显线索，也把动作写得更具体，例如不是“触碰贴纸”，而是“揭下兔毛贴纸的一角看看下面压着什么”。
- 如果某个物品只适合远观、不适合操作，才只给观察类动作；否则不要把 3-5 个 actions 全部写成“查看 / 观察 / 端详”的近义重复。
- 如果该物品当前没有合理互动方式，可以留空 actions 或将 interactable 设为 false。
- scene_changes 只在场景有变化时填写，包括角色移动、地点切换、时间推进、环境变化、在场 NPC 变化等。
- quest_updates 只在新任务或任务进度变化时填写。
- new_events 记录这段对话中的重要事件和造成的状态变更。
